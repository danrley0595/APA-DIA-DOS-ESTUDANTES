01 · Tokens — CSS Custom Properties

Todos os valores do sistema vivem em :root como variáveis CSS. Use-os diretamente no seu CSS ou em estilos inline. Nunca escreva valores fixos — sempre referencie um token.

ℹ️

Convenção de nomenclatura

--categoria-escala → ex: --color-blue-500, --space-4, --text-lg

Paleta — conceito de cores

A paleta combina três famílias com papéis distintos: azul para confiança e autoridade institucional (referência a saúde pública), ciano para conscientização e prevenção (Purple Ribbon — cor universal de prevenção ao abuso de substâncias), e âmbar para atenção e dados críticos sem alarmar.

02 · Tipografia

Os elementos h1–h6 e p já estão estilizados globalmente. Use tags semânticas corretas — o CSS cuida da aparência. Classes utilitárias complementam papéis específicos.

Headings — tag direta

<h1>Título principal da página</h1>    <!-- Sora ExtraBold 48px -->
<h2>Título de seção</h2>               <!-- Sora Bold 36px       -->

<h5>Rótulo agrupador</h5>             <!-- Sora SemiBold 20px   -->
<h6>Rótulo menor</h6>                 <!-- Sora Medium 18px     -->

03 · Botões

Todo botão começa com .btn. Combine com um modificador de variante e opcionalmente um de tamanho. A base cuida de layout, foco, transição e estado desabilitado.

Variantes

<button class="btn btn--primary">Primário</button>
<!-- Azul institucional. Ação principal — uma por contexto. -->

<button class="btn btn--secondary">Secundário</button>
<!-- Outline azul. Ação secundária ou alternativa. -->

<button class="btn btn--accent">Acento</button>
<!-- Ciano. CTA em hero ou destaque de conscientização. -->

<button class="btn btn--ghost">Ghost</button>
<!-- Sem preenchimento. Ações terciárias, cancelar. -->

<button class="btn btn--danger">Destrutivo</button>
<!-- Vermelho. Excluir, ação irreversível. -->

<button class="btn btn--dark">Dark</button>
<!-- Azul-900. Use sobre fundos azuis escuros. -->

Tamanhos

<button class="btn btn--primary btn--sm">Pequeno</button>
<button class="btn btn--primary">Médio (padrão)</button>
<button class="btn btn--primary btn--lg">Grande</button>
<button class="btn btn--primary btn--xl">Extra grande</button>
<button class="btn btn--primary btn--full">Largura total</button>

Estados e uso como link

<!-- Desabilitado: atributo nativo — opacidade 45%, sem cursor -->
<button class="btn btn--primary" disabled>Desabilitado</button>

<!-- Como link de navegação -->
<a href="/pagina" class="btn btn--primary">Ir para página</a>
<a href="/arquivo.pdf" class="btn btn--secondary btn--sm">Baixar PDF</a>

04 · Formulários e Inputs

Envolva cada campo em .form-group. Use .form-label + campo + .form-hint opcional. O foco azul e o anel âmbar de acessibilidade são automáticos.

Estrutura de um campo

<div class="form-group">
  <label class="form-label">
    Nome do campo <span class="required">*</span>
  </label>
  <input class="form-input" type="text" placeholder="Escreva aqui" />
  <span class="form-hint">Texto de ajuda opcional.</span>
</div>

06 · Abas (Tabs)

Tabs são CSS + JS mínimo incluso no rodapé. O JS switchTab(groupId, triggerEl, panelId) move a classe .is-active entre triggers e painéis.

Estrutura HTML obrigatória

<!-- 1. Wrapper com id único por grupo -->
<div class="tabs" id="meu-grupo">

  <!-- 2. Lista de triggers -->
  <div class="tabs__list" role="tablist">
    <button class="tabs__trigger is-active"
      onclick="switchTab('meu-grupo', this, 'painel-a')">
      Aba 1
    </button>
    <button class="tabs__trigger"
      onclick="switchTab('meu-grupo', this, 'painel-b')">
      Aba 2
    </button>
  </div>

  <!-- 3. Painéis — id = terceiro arg do switchTab -->
  <div id="painel-a" class="tabs__panel is-active">
    Conteúdo da Aba 1
  </div>
  <div id="painel-b" class="tabs__panel">
    Conteúdo da Aba 2
  </div>
</div>

Variante padrão — underline azul

Conteúdo da primeira aba. A classe .is-active no trigger acende o underline azul e exibe o painel.

Conteúdo da segunda aba. Pode ter qualquer markup: tabelas, cards, formulários.

Conteúdo da terceira aba. O .tabs__list faz scroll horizontal automaticamente em mobile.

Variante pill — .tabs--pill

<div class="tabs tabs--pill" id="meu-grupo">...</div>

Pill: fundo cinza quente, ativa fica branca com sombra. Ideal para filtros e alternadores curtos.

Sem borda inferior — visual encapsulado, diferente do underline padrão.

Terceira opção do grupo pill.

Variante dark — .tabs--dark

<div class="tabs tabs--dark" id="meu-grupo">...</div>
<!-- Cabeçalho azul-900, aba ativa em âmbar-300 -->

Painel escuro. Use esta variante dentro de .section--dark ou em cards com .card--dark.

Segundo painel do tema dark.

Terceiro painel do tema dark.
